## 设计决策备忘

1. **名称→代码匹配不用 LLM**：akshare 的 fund_name_em 返回全市场 26944 只基金的全量表（模块级缓存），首屏加载 1-2 秒后 O(1) 查表，比 LLM 快且免费。

2. **ETF 过滤逻辑**：akshare 的 31 种基金类型中不存在"ETF-场内"值，ETF 和联接基金同类型（如"指数型-股票"）。因此用名称过滤：含"ETF"但无"联接/连接"→跳过。

3. **导入不阻塞基金规则同步**：_resolve_fund_code 只做轻量的名称→代码查找 + 写最小 FundRule（fund_type 已知）。完整费率/确认日同步留给后台或用户手动触发，保证导入秒出候选。

4. **份额 2 位小数**：公募基金份额精确到 0.01 份，后端 round(x, 2)，前端输入 step="0.01"。

5. **SQLite WAL**：WAL 模式允许多读一写并发，解决双图同时上传时的锁冲突。busy_timeout=5000ms 作为兜底。
